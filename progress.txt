## US-001: Create lead API endpoint
**What I did:** Created POST /api/admin/leads endpoint that accepts { name, email, company, role, data_needs, use_case, admin_notes }, validates required fields (name, email) and email format via regex, inserts into Supabase leads table with status 'pending', returns 409 on duplicate email (error code 23505), and returns 201 with the created lead object. Uses the same cookie-based admin auth pattern (verifyAdminToken) as other admin routes.
**Files changed:** src/app/api/admin/leads/route.ts (new), prd.json (US-001 passes: true), progress.txt
**Learnings:** The leads table has a UNIQUE constraint on email, and Supabase surfaces PostgreSQL error code 23505 for unique violations. The existing admin routes all follow the same auth pattern: read admin-token cookie, verify via verifyAdminToken, return 401 if invalid. The insert data is built incrementally with optional fields only included if they are non-empty strings.

## US-004: Branded invite email template
**What I did:** Created src/lib/email/invite.ts with an exported sendInviteEmail({ to, name, magicLink }) function. Uses the Resend SDK with RESEND_API_KEY env var (returns { success: false, error: "Email not configured" } if missing). From address uses RESEND_FROM_EMAIL with fallback to team@claru.ai, formatted as "Claru AI <address>". Subject: "Your Claru Data Portal Access". HTML template uses the terminal aesthetic with dark background (#050505), accent green (#00ff88), JetBrains Mono font. Body includes personalized greeting, welcome message about portal access, prominent "Access Portal" button with the magic link, 24-hour expiry note, and support contact (team@claru.ai). Includes plain text fallback. Function never throws -- all errors are caught and returned as { success: false, error: string }.
**Files changed:** src/lib/email/invite.ts (new), prd.json (US-004 passes: true), progress.txt
**Learnings:** The existing email templates in src/lib/email.ts use slightly different colors (#0a0908 bg, #92B090 accent) than the landing page design system (#050505, #00ff88). The PRD explicitly requested the landing page colors for this template, creating a more terminal-aesthetic feel. The Resend SDK accepts a `text` property alongside `html` for plain text fallback. The function instantiates Resend inside the function body (after checking the API key) rather than at module scope to avoid errors when the env var is missing.
