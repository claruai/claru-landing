**Learnings:**
- PostgREST PGRST102 "All object keys must match": all JSON objects in a batch POST must have identical keys. Must include all fields (with null) even for optional ones like embedding.
- PostgREST `resolution=ignore-duplicates` does NOT work with expression-based unique indexes (COALESCE). Only works with actual UNIQUE constraints. Workaround: single-row inserts with 409 handling for dataset_clips.
- Caption text correctly built from agent_context.scene_summary (not .caption) plus environments, activities, objects, camera_perspective — matching agentContextToEmbeddingText() from openai.ts.
- 112 of 588 unique clips have embeddable text (from agent_context); remaining 476 have no caption text and will need the caption rebuild script (US-018) to derive captions from ann_metadata.
- Supabase MCP execute_sql runs as `postgres` role which is blocked by service_role-only RLS on clips/dataset_clips. Validation must use PostgREST REST API with service role key, not SQL editor.
