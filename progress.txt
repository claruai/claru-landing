**What I did:** Created POST /api/admin/leads endpoint that accepts { name, email, company, role, data_needs, use_case, admin_notes }, validates required fields (name, email) and email format via regex, inserts into Supabase leads table with status 'pending', returns 409 on duplicate email (error code 23505), and returns 201 with the created lead object. **Files changed:** src/app/api/admin/leads/route.ts (new), prd.json (US-001 passes: true), progress.txt
**Learnings:** The leads table has a UNIQUE constraint on email, and Supabase surfaces PostgreSQL error code 23505 for unique violations. The existing admin routes all follow the same auth pattern: read admin-token cookie, verify via verifyAdminToken, return 401 if invalid. The insert data is built incrementally with optional fields only included if they are non-empty strings.
