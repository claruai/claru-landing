## US-001: S3 media proxy endpoint
**What I did:** Verified the existing implementation at src/app/api/media/s3/route.ts. The file was already a complete implementation (not just a skeleton). It handles: GET /api/media/s3?key=KEY returning 302 redirect to a freshly signed S3 URL via getS3SignedUrl from presigner.ts with 1-hour expiry. Key validation rejects '..' (path traversal) and leading '/' (absolute paths). Response includes Cache-Control: private, max-age=3000. Returns 400 for missing key, 500 if signing fails.
**Files changed:** prd.json (US-001 passes: true), progress.txt
**Learnings:** The route was already fully implemented from a prior session. The presigner.ts supports both CloudFront signed URLs (preferred when configured) and S3 presigned URLs as fallback. The 302 redirect approach lets the browser handle range requests natively, which is critical for video seeking.

## US-002: S3 URL rewriting utility
**What I did:** Created src/lib/deck-builder/rewrite-s3-urls.ts with two exported functions. `rewriteS3ToProxy(html: string): string` is a synchronous function that rewrites S3 references in HTML to use /api/media/s3?key=KEY proxy URLs. Handles three patterns: (1) s3-placeholder.local/* URLs from the design agent, (2) moonvalley-annotation-platform.s3.*.amazonaws.com/KEY URLs (signed, unsigned, or double-signed -- extracts key before ?), (3) {{s3:key}} template tokens. Already-proxied URLs and non-S3 URLs are left unchanged. Fast path: returns early if none of the S3 markers are found in the HTML. Also created `extractMediaRefs(html: string): string[]` for US-011 (dependency tracking) which parses proxy URLs and returns unique S3 keys.
**Files changed:** src/lib/deck-builder/rewrite-s3-urls.ts (new), prd.json (US-002 passes: true), progress.txt
**Learnings:** The key design decision is that this function does NOT sign URLs -- it only rewrites them to the proxy pattern. The proxy endpoint (/api/media/s3) handles signing at request time, so URLs in the HTML never expire. The S3 keys are URL-encoded via encodeURIComponent when building proxy URLs to handle keys with spaces or special characters. The extractMediaRefs function uses decodeURIComponent to reverse the encoding when extracting keys.
