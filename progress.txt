## US-001: S3 media proxy endpoint
**What I did:** Verified the existing implementation at src/app/api/media/s3/route.ts. The file was already a complete implementation (not just a skeleton). It handles: GET /api/media/s3?key=KEY returning 302 redirect to a freshly signed S3 URL via getS3SignedUrl from presigner.ts with 1-hour expiry. Key validation rejects '..' (path traversal) and leading '/' (absolute paths). Response includes Cache-Control: private, max-age=3000. Returns 400 for missing key, 500 if signing fails.
**Files changed:** prd.json (US-001 passes: true), progress.txt
**Learnings:** The route was already fully implemented from a prior session. The presigner.ts supports both CloudFront signed URLs (preferred when configured) and S3 presigned URLs as fallback. The 302 redirect approach lets the browser handle range requests natively, which is critical for video seeking.
